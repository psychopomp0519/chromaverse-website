# Novel Verification Prompt Short

```text
소설 원고를 독자 감상형이 아니라 작가용 기준으로 검증해라.

목표:
- 현재 작성된 원고 범위를 읽고 작가용 피드백 작성
- 무엇이 작동하고 무엇이 약한지 구조적으로 진단
- 화별 기능, 카이 능동성, 문장 밀도, 캐릭터 축, 연재성, 상업성까지 포함

입력:
- 프로젝트 루트: <PROJECT_ROOT>
- 원고 폴더: <NOVEL_DIR>
- 출력 문서: <OUTPUT_DOC_PATH>
- 필요 시 이전 피드백: <PREV_FEEDBACK_PATH>

반드시 할 것:
1. 파일 상태 확인
- 화수 파일 목록
- 파일명/정렬
- 글자 수
- 변경 여부

2. 원고 전체 읽기
- 검증 대상 범위를 처음부터 끝까지 읽어라

3. 총평 작성
- 이전보다 좋아졌는지
- 가장 크게 개선된 점
- 가장 크게 남은 문제
- 가장 냉정한 한 줄 평가

4. 규칙 검증
- 분량
- 카운트다운/수치 정합성
- 캐릭터 보이스
- 용어 계층
- 화자 시점 규칙

5. 초반 장사력 검증
- 1화가 기능하는가
- 2화가 분위기만 남지 않는가
- 3화까지 재미 포인트가 보이는가

6. 화별 검증
- 이 화의 목적
- 이 화에서 바뀌는 것
- 강점
- 약점
- 현재 구조에서 필요한 이유

7. 캐릭터 검증
- 카이, 레이, 닉스, 엔 중심
- 말투, 감정 기능, 서사 기능, 기능 중복 여부

8. 문장 밀도 검증
- 의미심장한 문장 과밀
- 평문/완급 부족
- 좋은 문장 때문에 리듬이 죽는지

9. 현재 작성분 마지막 화 검증
- 다음 화를 보게 만드는가
- 과하게 닫히지 않는가
- 현재까지 쌓은 축을 유지하는가

반드시 같이 볼 것:
- 매 화 경량 검증: 분량, 보이스, 용어, 카운트다운, 카이 능동성, 화말 훅
- 3화 단위 검증: 사건 전진, 감정 곡선, 설명 대 사건 비율, 훅 분포, 이탈 가능성
- 12화 단위 검증: 구조, 캐릭터 축, 떡밥/회수, 미스터리 공개 속도, 다음 12화 연결력

출력 문서 구조:
1. 제목
2. 검토 대상
3. 총평
4. 가장 크게 개선된 점
5. 아직 남은 가장 큰 문제
6. 규칙 검증
7. 화별 검증
8. 캐릭터별 검증
9. 연재성/상업성 판단
10. 최종 한 줄 결론

최종 응답:
- 문서 저장 경로
- 가장 큰 개선 2~3개
- 가장 큰 남은 문제 2~3개
- 필요하면 즉시 손봐야 할 화 1개 지정

이제 검증을 시작해라.
```
