# Novel Verification Playbook

## 목적

이 문서는 이 세션에서 실제로 수행한 `소설 원고 검증 방식`을 다음에도 동일하게 반복할 수 있도록 정리한 운영 문서다.  
핵심 목적은 다음과 같다.

- 작가용 기준으로 원고를 검증한다
- 독자 감상이 아니라 구조와 기능을 본다
- 화 단위, 3화 단위, 12화 단위로 나눠 품질을 유지한다
- 피드백이 바로 수정 방향으로 이어지게 만든다

## 기본 원칙

1. 기본 출력은 `작가용`
- 독자 감상문보다 구조적 진단을 우선한다.

2. 칭찬보다 진단
- 무엇이 강한지 쓰되, 무엇이 약한지와 왜 약한지까지 반드시 적는다.

3. 화자성과 구조를 같이 본다
- 문체만 보거나 플롯만 보지 않는다.

4. “잘 썼다”와 “잘 읽힌다”를 구분한다
- 둘을 혼동하지 않는다.

## 검증 단위

## 1. 매 화 경량 검증

목적:

- 기본 품질 붕괴 방지

확인 항목:

- 분량 규칙
- 캐릭터 보이스
- 용어 정합성
- 카운트다운/수치 연속성
- 카이 능동성 1회 이상
- 화말 훅 존재

## 2. 3화 단위 검증

목적:

- 리듬 유지
- 독자 이탈 방지

반드시 돌릴 문서:

- `project-assets/feedback/크로마버스_3화단위_검증_체크리스트.md`

## 3. 12화 단위 정식 검증

목적:

- 구조 유지
- 캐릭터/미스터리/떡밥/연재성 점검

반드시 돌릴 문서:

- `project-assets/feedback/크로마버스_12화단위_정식검증_체크리스트.md`

## 실제 실행 순서

## 1. 원고 파일 상태 확인

먼저 확인:

- 현재 화수 파일 목록
- 파일명/화수 정렬
- 글자 수
- 이전 버전 대비 변경 여부

반드시 체크:

- 분량 규칙 위반 화
- 제목 변경 여부
- 화수 순서 변경 여부

## 2. 원고 전체 읽기

검증 대상 범위 전체를 다시 읽는다.

현재 세션 기준으로는:

- `1화~12화`

원칙:

- 발췌로 판단하지 않는다
- 초반/중반/현재 작성분 마지막 화까지 흐름으로 읽는다

## 3. 먼저 총평을 잡는다

읽고 나서 먼저 아래를 적는다.

- 지금 버전이 이전보다 좋아졌는가
- 가장 크게 개선된 것
- 가장 크게 남은 것
- 지금 원고의 가장 냉정한 한 줄 평가

## 4. 규칙 준수 검증

반드시 본다.

- 분량 규칙
- 카운트다운/숫자 정합성
- 캐릭터 보이스
- 용어 계층 규칙
- 화자 시점 규칙

## 5. 초반 장사력 검증

특히 1~3화는 별도로 본다.

질문:

- 1화가 기능하는가
- 2화가 분위기화에 머무르지 않는가
- 3화까지 읽었을 때 작품의 재미 포인트가 보이는가
- “좋은데 느리다”는 평가를 얼마나 줄였는가

## 6. 화별 기능 검증

각 화마다 반드시 적는다.

- 이 화의 목적
- 이 화에서 바뀌는 것
- 이 화의 강점
- 이 화의 약점
- 이 화가 현재 구조에서 필요한 이유

## 7. 캐릭터 축 검증

주요 인물별로 본다.

- 카이
- 레이
- 닉스
- 엔

체크:

- 말투 유지
- 기능 중복 여부
- 감정 기능
- 서사 기능
- 다음 화 확장성

## 8. 문장 밀도 검증

반드시 본다.

- 의미심장한 문장 과밀 여부
- 평문/완급 부족 여부
- 강한 문장이 강한 장면에 집중되는지
- “좋은 문장” 때문에 리듬이 죽는 구간이 있는지

## 9. 현재 작성분 마지막 화 검증

중요:

- 현재 마지막 화를 `완결`처럼 보지 않는다
- `현재 작성분의 끊는 지점`으로만 본다

질문:

- 다음 화를 보게 만드는가
- 지금까지 쌓은 축을 유지한 채 넘기는가
- 과하게 닫히거나 너무 허무하게 끊기지 않는가

## 문서 작성 방식

기본 문서 유형 3개:

1. `작가용 직설 피드백`
- 구조/강점/약점/상업성/이탈 지점

2. `작가용 검증 리포트`
- 이전 피드백 반영 후 재검증

3. `체크리스트`
- 3화용 / 12화용

권장 파일명 예시:

- `project-assets/feedback/크로마버스_1화-12화_작가용_직설피드백_YYYY-MM-DD.md`
- `project-assets/feedback/크로마버스_1화-12화_재검증_작가용_YYYY-MM-DD.md`

## 작가용 문서에 반드시 들어갈 항목

1. 총평
2. 가장 크게 개선된 점
3. 아직 남은 가장 큰 문제
4. 규칙 준수 여부
5. 화별 검증
6. 캐릭터별 검증
7. 상업성/연재성 판단
8. 최종 한 줄 결론

## 핵심 평가 기준

## 1. 사건 전진

매 화 사건이 전진해야 한다.  
설명/분위기만으로 끝나는 화는 위험 신호다.

## 2. 카이 능동성

카이는 관찰자지만, 수동 주인공이 되면 안 된다.  
매 화 최소 1회는 결과를 바꾸는 선택이 보여야 한다.

## 3. 문장 완급

모든 문장이 무거우면 진짜 무거운 문장이 묻힌다.  
좋은 문장을 줄이는 게 아니라, 배치 밀도를 조절해야 한다.

## 4. 독자 체류 논리

반드시 본다.

- 왜 1화를 넘기나
- 왜 3화를 넘기나
- 왜 5화를 결제하나
- 왜 12화까지 남아 있나

## 5. 기능화 방지

특히 체크:

- 레이가 단순 희생 장치가 되는지
- 닉스가 단순 폭주 장치가 되는지
- 엔이 단순 완충재가 되는지

## 이번 세션에서 실제로 확인한 운영 교훈

1. `독자용`보다 `작가용`이 더 유용했다
- 이후 기본 모드는 작가용으로 유지

2. `1~12화`처럼 누적 분량 재검증이 중요하다
- 한 번 좋았던 평가를 유지하는지 다시 봐야 한다

3. `현재 작성분 마지막 화`라는 표현이 중요하다
- 완결처럼 읽으면 판단이 어긋난다

4. 분량 규칙은 실제로 유효했다
- 글자 수가 튀는 화는 실제 체감도 흔들리는 경우가 많았다

5. 3화/12화 단위 검증이 가장 현실적이다
- 매 화 검증은 붕괴 방지
- 3화는 리듬 유지
- 12화는 구조 유지
