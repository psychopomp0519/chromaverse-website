# 서브플롯과 다선 구조

> 메인플롯은 뼈대, 서브플롯은 살이다.
> 단선 서사는 100화가 한계, 다선 서사는 1,000화도 가능.

---

## 1. 서브플롯의 역할

### 1.1 왜 서브플롯이 필요한가

| 역할 | 설명 |
|------|------|
| **깊이** | 주인공의 다면성, 세계의 입체감 확보 |
| **페이싱** | 메인플롯의 긴장↔이완 조절 장치 |
| **주제 강화** | 다른 각도에서 같은 테마를 반복 변주 |
| **독자 다양성** | 다양한 취향의 독자를 동시에 만족 |
| **예측 방지** | 독자의 예측을 빗나가게 하는 우회로 |
| **연재 안정성** | 메인 아크 사이 공백을 서브플롯이 채움 |

### 1.2 서브플롯 유형

| 유형 | 설명 | 메인플롯과의 관계 |
|------|------|------------------|
| **거울형** | 메인과 같은 주제를 다른 인물/상황으로 | 주제 심화 |
| **대비형** | 메인과 정반대 결론을 향해 진행 | 주제 입체화 |
| **보완형** | 메인이 다루지 않는 영역을 커버 | 세계관/캐릭터 확장 |
| **합류형** | 독립적으로 진행되다 메인과 합류 | 서사 복잡도 증가 |
| **장식형** | 메인과 약한 연결, 분위기/세계관 용 | 밀도 증가 (주의: 과다 시 산만) |
| **타이머형** | 독립적으로 진행되며 메인에 시간 압박 | 긴장감 증폭 |

### 1.3 서브플롯 설계 규칙

**[필수]** 5가지 규칙:

1. **귀결 원칙**: 모든 서브플롯은 반드시 귀결(해소/합류/종료)되어야 함
2. **주제 연결**: 메인 테마와 최소 간접적으로 연결 (완전 무관한 서브플롯은 산만)
3. **비율 관리**: 서브플롯 합산이 전체 분량의 30~40% 이하 (초과 시 메인 희석)
4. **단계적 도입**: 1권에서는 메인+1서브, 이후 점진적 추가
5. **동시 가동 한계**: 독자가 추적 가능한 서브플롯은 동시 3~4개가 상한

**[비인기]** 실패 패턴:
- 서브플롯이 메인보다 재미있어서 독자가 메인을 건너뜀
- 서브플롯을 열어만 놓고 회수하지 않음 → 떡밥 부채
- 서브플롯 인물이 갑자기 소멸 → 독자 불만
- 서브플롯이 5개 이상 동시 진행 → 추적 불가

## 2. 서브플롯 운용 전략

### 2.1 도입과 종료 타이밍

```
메인 ─────────────────────────────────────→
서브A   ╭───────────────╮
서브B        ╭────────────────────╮
서브C              ╭──────────╮
         ^도입     ^교차      ^합류/종료
```

**도입 시점**:
- 메인플롯이 안정기에 들어갈 때 (긴장 공백 채우기)
- 새 아크 시작 시 (새 환경/인물과 함께)
- 메인 사건의 부산물로 자연스럽게

**종료 시점**:
- 메인 아크 클라이맥스 직전 (서브 정리 → 메인 집중)
- 메인플롯과 합류하며 자연 소멸
- 최종 아크에서 모든 서브가 메인에 기여하며 마감

### 2.2 교차 기법

| 기법 | 설명 | 효과 |
|------|------|------|
| **교대 배치** | A → B → A → B 장면 교대 | 두 줄기 긴장감 동시 유지 |
| **충돌** | 서브 인물이 메인 인물과 직접 부딪힘 | 복잡도 급상승 |
| **정보 비대칭** | 서브 장면에서 독자만 아는 정보 공개 | 서스펜스 강화 |
| **테마 에코** | 같은 화에서 메인/서브가 같은 주제를 다른 각도로 | 주제 강화 |
| **타임라인 수렴** | 별도 시간선이 한 지점에서 만남 | 카타르시스 |

### 2.3 장편(100화+)에서의 서브플롯 관리

| 화수 구간 | 서브플롯 전략 |
|----------|-------------|
| 1~20화 | 메인 확립, 서브A 씨앗 뿌리기만 |
| 20~50화 | 서브A 본격화 + 서브B 도입 |
| 50~100화 | 서브A 1차 귀결, 서브B/C 활성 |
| 100~200화 | 서브 합류 시작, 메인과 연결 강화 |
| 200화+ | 모든 서브가 메인에 기여하며 수렴 |

## 3. 다선(다시점) 구조

### 3.1 다선 구조 유형

| 유형 | 설명 | 예시 | 난이도 |
|------|------|------|--------|
| **단일 시점 + 서브 삽입** | 주인공 1인칭이지만 간헐적 타인 시점 | 대부분의 웹소설 | 낮음 |
| **교대 시점** | 2~3인물이 번갈아 주인공 | 연인 쌍방 시점 로맨스 | 중간 |
| **군상극** | 4+ 인물이 각자의 서사를 진행 | 전쟁/정치물 | 높음 |
| **시간 교차** | 같은 인물의 과거/현재 교대 | 회고록, 미스터리 | 중간 |
| **병렬 세계** | 서로 다른 세계/차원의 이야기가 병행 | SF, 이세계물 | 높음 |

### 3.2 다시점 서술의 규칙

**[필수]** 핵심 5원칙:

1. **시점 표지**: 시점 전환 시 독자가 즉시 "누구인지" 파악 (이름/상황/말투)
2. **균형**: 시점 인물별 분량 비율 사전 설계 (주인공 60% / 서브 각 20% 등)
3. **목적**: 각 시점이 "이 인물에서만 볼 수 있는 것"을 제공해야 함
4. **연결**: 시점 전환이 전체 서사에 기여 (단순 시점 변경은 산만)
5. **빈도**: 1회차 내 시점 전환은 2회 이하 (웹소설 기준)

**[금기]** 헤드호핑(Head-hopping):
- 같은 장면 내에서 시점이 무의식적으로 왔다갔다 하는 것
- "카이는 화가 났다. 코다는 그런 카이가 걱정되었다." → 1문단 내 시점 2회 전환
- 해결: 장면 단위로 시점 고정, 전환 시 명확한 표지

### 3.3 다선 구조의 수렴

**수렴 방법**:
| 방법 | 설명 |
|------|------|
| 물리적 합류 | 떨어져 있던 인물들이 같은 장소에 모임 |
| 정보 합류 | 각 시점에서 조각난 정보가 합쳐져 전체 그림 완성 |
| 주제 합류 | 각 서사가 같은 주제에 대해 다른 답을 내놓음 |
| 갈등 합류 | 개별 갈등이 하나의 대갈등으로 수렴 |
| 시간 합류 | 다른 시간대의 서사가 한 시점에서 만남 |

**[인기]** 수렴 순간 = 작품 최대의 카타르시스 포인트

## 4. 실전 워크시트

### 4.1 서브플롯 기획표

```
서브플롯 이름:
유형: [거울/대비/보완/합류/장식/타이머]
핵심 인물:
메인 테마와의 연결:
도입 화수:        귀결 화수:
분량 비중: ___%
메인플롯과의 교차 지점:
  - 1차 교차: __화
  - 2차 교차: __화
  - 합류/귀결: __화
```

### 4.2 다선 구조 밸런스 체크

```
시점 인물  | 분량 비율 | 장면 수 | 독자적 정보 | 합류 시점
---------|----------|---------|-----------|--------
인물 A    | 60%      | 120개   | 비밀 X, Y  | -
인물 B    | 25%      | 50개    | 비밀 Z     | 150화
인물 C    | 15%      | 30개    | 적 내부 정보 | 200화
```
