# 페이싱과 리듬

> 이야기의 속도감과 호흡. 같은 플롯도 페이싱에 따라 긴장감 넘치거나 지루해진다.
> 장면 배치, 텐션 조절, 호흡 관리의 실전 기법.

---

## 1. 페이싱의 기본 원리

### 1.1 텐션 파형 모델

```
높음 ████████
         ████████
     ████        ████████
████                    ████████
                                ████
낮음 ─────────────────────────────────→ 시간
     도입    상승    절정    이완  여운
```

**핵심**: 텐션은 "계속 올라가는 것"이 아니라 **올렸다 내렸다의 파동**

**[인기]** 파동 리듬:
- 고텐션 2~3회 → 저텐션 1회 (3:1 비율)
- 저텐션 구간에서 캐릭터 심화/세계관 확장/유머
- 각 고텐션 피크가 이전보다 높아야 함

**[비인기]** 실패 패턴:
- 일직선 상승: 독자가 지침 → "쉴 틈이 없다"
- 일직선 수평: 아무 일도 안 일어남 → "재미없다"
- 불규칙 난잡: 올렸다 급격히 떨어지고 또 올림 → "뭐야 이 롤러코스터"

---

### 1.2 장면 속도의 3요소

| 요소 | 빠르게 하려면 | 느리게 하려면 |
|------|-------------|-------------|
| **문장 길이** | 짧은 단문, 3~5어절 | 긴 복문, 묘사 삽입 |
| **장면 밀도** | 행동 중심, 대사 위주 | 내면 묘사, 감각 묘사 |
| **시간 흐름** | 요약 서술, 시간 점프 | 실시간 서술, 순간 확대 |

**[팁]** 한 장면 안에서도 속도를 변주할 수 있음:
- 전투 중 "슬로모션" → 결정적 일격의 순간을 느리게
- 일상 중 "타임랩스" → 반복적 일과를 한 문장으로 압축

## 2. 장면 배치 전략

### 2.1 장면 유형별 속도감

| 장면 유형 | 속도 | 역할 | 배치 위치 |
|----------|------|------|----------|
| 액션/전투 | 매우 빠름 | 긴장, 카타르시스 | 클라이맥스, 전환점 |
| 대화/논쟁 | 빠름~보통 | 정보 전달, 관계 변화 | 자유 배치 |
| 탐색/이동 | 보통 | 세계관, 분위기 | 전환 구간 |
| 내면 독백 | 느림 | 캐릭터 깊이, 감정 | 결정 전/후 |
| 묘사/풍경 | 매우 느림 | 분위기, 여운 | 도입, 이완 구간 |
| 회상 | 느림~보통 | 배경, 동기 부여 | 위기 전/후 |

### 2.2 장면 시퀀스 패턴

**패턴 A: 가속 시퀀스** (클라이맥스 향해)
```
묘사(느림) → 대화(보통) → 갈등 발생(빠름) → 추격/전투(매우 빠름)
```

**패턴 B: 감속 시퀀스** (클라이맥스 후)
```
전투 종료(빠름) → 결과 확인(보통) → 감정 정리(느림) → 풍경/여운(매우 느림)
```

**패턴 C: 파도 시퀀스** (연재 1회분)
```
훅(빠름) → 상황 설명(보통) → 위기 고조(빠름) → 잠깐 이완(느림)
→ 재점화(빠름) → 클리프행어(빠름)
```

---

### 2.3 연재 웹소설의 페이싱 공식

**1회차 내부 리듬** (3,000~5,000자 기준):

```
도입 훅 (200~300자): 긴장감 있는 장면/대사/상황으로 시작
  ↓
상황 전개 (800~1,200자): 배경 설명 + 캐릭터 행동
  ↓
중간 포인트 (500~800자): 작은 반전 또는 정보 공개
  ↓
상승 전개 (800~1,200자): 갈등 심화, 스테이크 상승
  ↓
클리프행어 (200~300자): 다음 회차로 이어지는 절단
```

**5~10회 아크 리듬**:

| 회차 | 역할 | 텐션 |
|------|------|------|
| 1화 | 아크 시작, 목표 설정 | ▲ 중 |
| 2화 | 준비/탐색, 세력 파악 | ▽ 낮음~중 |
| 3화 | 첫 번째 시도, 작은 성공 | ▲ 중 |
| 4화 | 반격, 상황 역전 | ▲▲ 높음 |
| 5화 | 위기, 가장 어두운 순간 | ▲▲▲ 최고 |
| 6화 | 전환점, 해결의 실마리 | ▽ 중 |
| 7화 | 반격 준비/결의 | ▲ 중~높음 |
| 8화 | 클라이맥스, 대결 | ▲▲▲ 최고 |
| 9화 | 해결, 보상 | ▽ 낮음 |
| 10화 | 여운 + 다음 아크 암시 | ▲ 중 (훅) |

## 3. 속도 조절 기법

### 3.1 가속 기법

**단문 연쇄**:
```
달렸다. 숨이 찼다. 뒤돌아볼 수 없었다.
발이 미끄러졌다. 일어났다. 다시 달렸다.
```

**행동 동사 나열**:
```
검을 뽑고, 앞으로 내질렀다. 막혔다. 빗겨냈다. 회전. 다시 찌른다.
```

**대사 연타**:
```
"거기 서."
"왜."
"이유를 알잖아."
"모르겠는데."
"거짓말."
```

**시간 압축**:
```
3일이 지났다. 상황은 나아지지 않았다.
```

### 3.2 감속 기법

**감각 묘사 확장**:
```
바람이 뺨을 스쳤다. 차갑지만 날카롭지는 않은, 봄이 오기 직전의 그런 바람이었다.
어딘가에서 젖은 흙 냄새가 났다. 눈이 녹고 있는 것이다.
```

**내면 독백 삽입**:
```
그때 나는 알았어야 했다. 이 평화가 오래가지 않을 거라는 것을.
하지만 그 순간의 나는 그저 — 따뜻한 국물의 온기에 취해 있었다.
```

**행동의 미세 분해**:
```
그가 컵을 집어 들었다. 잠시 입술에 대었다가, 뜨거운지 미간을 찡그렸다.
한 모금. 그리고 긴 한숨.
```

**풍경 전환**:
```
전투가 끝난 들판 위로 노을이 내려앉고 있었다.
피와 흙이 뒤섞인 대지를 붉은 빛이 물들였다.
기묘하게도 — 아름다웠다.
```

## 4. 장르별 페이싱 특성

### 4.1 판타지/액션
- 전투 : 일상 = 4 : 6 (전투 과다 시 피로)
- 전투 장면 내부: 짧은 단문 70% + 묘사 30%
- 레벨업/보상 장면은 "이완+쾌감"의 이중 역할

### 4.2 로맨스
- 밀당 리듬: 가까워지기(기대) → 멀어지기(불안) → 더 가까워지기(안도)
- 심장이 뛰는 장면: 느리게 (순간 확대, 감각 집중)
- 오해/갈등 장면: 빠르게 (감정적 충돌)

### 4.3 미스터리/스릴러
- 단서 발견: 느리게 (독자에게 추리 시간)
- 추격/위험: 매우 빠르게
- 반전: "정지 + 폭발" (갑자기 멈추고 → 충격적 정보)

### 4.4 일상/힐링물
- 전반적으로 느린 템포가 기본
- 소소한 사건으로 미세한 파동 생성
- 감정의 진폭은 작지만 빈도는 높게

### 4.5 호러
- 점진적 가속 → 급정지 → 충격 (Jump Scare 문학 버전)
- 고요한 장면이 길수록 다음 공포가 강력
- **[팁]**: "아무 일도 안 일어나는 불안"이 가장 무서움

## 5. 페이싱 실패 진단

| 독자 반응 | 원인 | 처방 |
|----------|------|------|
| "지루하다" | 저텐션 구간이 너무 김 | 작은 갈등/유머/정보 삽입 |
| "피곤하다" | 고텐션 연속, 이완 없음 | 캐릭터 호흡 장면 삽입 |
| "뭔가 빠르다" | 감정 정리 없이 다음 사건 | 리액션 장면 추가 |
| "느리다" | 묘사/독백 과다 | 행동/대화로 정보 전달 |
| "산만하다" | 장면 전환이 잦고 연결 부재 | 장면 간 다리(Bridge) 강화 |
| "갑작스럽다" | 에스컬레이션 단계 생략 | 중간 단계 보충 |

## 6. 페이싱 설계 도구

### 6.1 텐션 그래프 (직접 그려보기)

```
Y축: 텐션 레벨 (1~10)
X축: 회차/장/장면 번호

각 장면의 텐션 레벨을 점으로 찍고 연결
→ 건강한 그래프: 들쑥날쑥하되 전반적 우상향
→ 문제 그래프: 수평선, 급강하, 지속적 최고텐션
```

### 6.2 장면 카드 기법

각 장면을 카드로 만들어 배치:
```
[장면명] | [텐션: 높/중/낮] | [속도: 빠름/보통/느림] | [역할: 행동/감정/정보]
```

→ 카드를 늘어놓고 "높-높-높"이 3장 연속이면 사이에 "낮" 삽입
→ "느림-느림-느림"이 3장 연속이면 하나를 "빠름"으로 교체

### 6.3 호흡 체크 (완성 후 검증)

1. 원고를 소리 내어 읽으며 "숨이 막히는 부분"과 "졸리는 부분" 표시
2. 숨 막히는 부분: 이완 장면 삽입 검토
3. 졸리는 부분: 갈등/반전/유머 삽입 검토
4. 두 표시가 교대로 나타나면 건강한 페이싱
