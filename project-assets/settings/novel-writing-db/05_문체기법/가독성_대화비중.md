# 가독성과 대화 비중

> 웹소설의 가독성은 문학적 완성도와 별개의 기술.
> "술술 읽히는" 글의 비밀: 대화 비중, 문단 구성, 시각적 리듬.

---

## 1. 가독성의 핵심 요소

### 1.1 웹소설 가독성 ≠ 문학적 문장력

| 문학적 문장력 | 웹소설 가독성 |
|-------------|-------------|
| 문장의 아름다움 | 읽는 속도의 쾌적함 |
| 복문, 다층 구조 | 짧고 명확한 구조 |
| 여운, 행간의 의미 | 직접적 전달 |
| 재독 시 새로운 발견 | 첫 독에서 즉각 이해 |

**[필수]** 웹소설은 "모바일에서 스크롤하며 읽는" 환경:
- 작은 화면, 짧은 집중 시간
- 한 문단이 화면 절반 이상 차지하면 "벽" 느낌 → 스킵
- 대화가 없이 서술만 이어지면 "무거운" 느낌

---

## 2. 대화 비중 관리

### 2.1 이상적 대화 비중

| 매체 | 대화 비중 | 이유 |
|------|----------|------|
| 웹소설 | **50~70%** | 가독성, 속도감, 캐릭터 표현 |
| 단행본 장편 | 30~50% | 묘사와 서술의 여유 |
| 문학 소설 | 20~40% | 언어적 탐구, 내면 묘사 |
| 단편 | 자유 | 효과에 따라 |

### 2.2 "대화만 읽어도 이해되는가?" 테스트

**[인기]** 웹소설 핵심 원칙:
> 대화만 골라 읽어도 줄거리를 따라갈 수 있어야 한다

```
❌ 나쁜 예 (대화 없이 읽으면 이해 불가):
서술: 그는 중요한 결정을 내렸다.

✅ 좋은 예 (대화가 정보를 전달):
"내일 출발한다. 더 기다릴 수 없어."
"혼자?"
"...혼자."
→ 출발, 급한 사정, 혼자 가는 것에 대한 갈등이 대화만으로 전달
```

### 2.3 대화와 지문의 배치

**빈도**: 서술 3~4문장 → 대화 삽입 → 서술 2~3문장 → 대화

**지문의 역할**: 대사 사이에 행동/감정/풍경을 삽입하여 시각적 리듬 생성

```
"거기 서." 그가 검을 뽑았다.
"왜?"
바람이 불었다. 그의 외투 자락이 펄럭였다.
"이유를 알잖아."
```
→ 대사-지문-대사-지문의 리듬이 읽기 쾌적

## 3. 문단 구성

### 3.1 웹소설 문단 길이

**[필수]** 한 문단 = 2~4문장 (모바일 기준 4~8줄)

```
❌ 벽 문단 (10줄 이상):
독자가 스크롤하다가 "긴 문단 = 스킵" 반사 발생

✅ 호흡 있는 문단:
2~3문장 서술.

한 줄 여백.

다시 2~3문장.
```

### 3.2 시각적 여백의 힘

- 빈 줄(여백)은 "쉼표"의 역할
- 충격적 정보 전후에 여백 → 효과 증폭
- 장면 전환 시 여백 → 자연스러운 전환

```
그가 미소 지었다.

— 그 미소가 마지막이었다.
```

### 3.3 한 줄 강조

| 기법 | 예시 | 효과 |
|------|------|------|
| 극단문 단독 배치 | "끝이었다." | 충격, 여운 |
| 대사 단독 배치 | "...거짓말." | 감정 집중 |
| 내면 단독 배치 | "알고 있었다, 처음부터." | 인식 전환 |

## 4. 읽기 속도 최적화

### 4.1 어려운 단어 사용 빈도

**원칙**: 전문용어/한자어/고어(古語)는 1페이지당 1~2개 이하
- 세계관 고유 용어도 포함
- 처음 등장 시 맥락으로 의미 파악 가능해야 함

### 4.2 문장 구조 단순화

```
❌ 복잡: "이전에 그가 말했던 것이 사실이었다면, 그리고 그것이 내가 생각하는 것과 같다면, 우리는 크게 착각하고 있었던 셈이다."

✅ 단순: "그의 말이 사실이라면. 우리는 처음부터 틀렸다."
```

### 4.3 접속사 절제

**과다 접속사** = 읽기 속도 저하:
```
❌ "그러나 한편으로는 그렇지만 또한 동시에"
✅ 접속사 없이 문장을 끊고 새로 시작
```

|--------|----------|------------|

| 문피아 | 긴 호흡 수용 | 대화 40%+, 서술 허용 |
| 노벨피아 | 다양 | 장르에 따라 가변 |

## 6. 가독성 자가 테스트

1. **스크롤 테스트**: 모바일로 자기 글을 읽어보기. 스크롤이 멈추는 곳 = 가독성 문제.
2. **대화 추출 테스트**: 대사만 뽑아 읽어보기. 이야기를 따라갈 수 있는가?
3. **소리 읽기 테스트**: 소리 내어 읽었을 때 숨이 막히는 곳 = 문장이 너무 긴 곳.
4. **벽 문단 체크**: 6줄 이상 이어지는 문단이 있는가? → 분할.
