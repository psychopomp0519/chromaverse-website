# 집필 프로세스/워크플로우

> 아이디어에서 완성 원고까지의 실전 단계.
> 기획→집필→퇴고의 전체 흐름.

---

## 1. 전체 프로세스

```
1. 아이디어 → 2. 기획 → 3. 설계 → 4. 초고 → 5. 퇴고 → 6. 완성
```

---

## 2. 1단계: 아이디어

### 2.1 아이디어 발굴법

| 방법 | 설명 |
|------|------|
| "만약에" 질문 | "만약 마법사가 마법을 쓸 수 없다면?" |
| 결합 | 기존 A + 기존 B = 새로운 C ("무협 + 회사물") |
| 역전 | 클리셰의 반대 ("용이 공주를 구하는 이야기") |
| 경험 | 자신의 감정/경험을 소재로 |
| 관찰 | 뉴스, 사람, 장소에서 영감 |

### 2.2 아이디어 검증

**3가지 질문**:
1. "이 아이디어로 200화를 쓸 수 있는가?" (연재 기준)
2. "이 아이디어를 한 줄(로그라인)로 설명할 수 있는가?"
3. "이 아이디어가 나만 쓸 수 있는 이야기인가?"

---

## 3. 2단계: 기획

### 3.1 기획 문서 구성

```
1. 로그라인
2. 장르/태그/타겟 독자
3. 테마 (핵심 질문)
4. 주인공 설정 (동기/결핍/성장 방향)
5. 핵심 조연 2~3명
6. 세계관 핵심 설정
7. 전체 줄거리 요약 (1페이지)
8. 아크 구성 (소/중/대 아크)
```

### 3.2 플롯 설계 방법

**탑다운 (Top-Down)**:
```
전체 결말 → 대 아크 → 중 아크 → 소 아크 → 회차
```
- 장점: 구조적, 복선 설계 용이
- 단점: 유연성 부족

**바텀업 (Bottom-Up)**:
```
핵심 장면 몇 개 → 장면 연결 → 아크 형성 → 전체 구조
```
- 장점: 창의적, 유기적
- 단점: 구조 약화 위험

**[팁]** 탑다운으로 뼈대 → 바텀업으로 살 붙이기 = 최적 조합

## 4. 3단계: 설계

### 4.1 회차별 플롯 설계

| 항목 | 내용 |
|------|------|
| 회차 번호/제목 | |
| 장면 목록 | 장면 1, 장면 2, ... |
| 핵심 사건 | 이 회차의 메인 이벤트 |
| 감정선 | 시작 감정 → 끝 감정 |
| 떡밥/복선 | 설치/회수 목록 |
| 훅/클리프 | 오프닝 훅, 엔딩 클리프 |

### 4.2 캐릭터 시트

```
이름:
외형 (인상적 특징 2~3개):
성격 (핵심 특성 3개):
동기/목표:
결핍/두려움:
거짓 신념(Lie):
성장 방향:
말투 특징:
관계 맵:
비밀:
```

## 5. 4단계: 초고

### 5.1 초고의 원칙

> "초고는 쓰레기여도 된다. 퇴고가 보석으로 만든다." — 헤밍웨이(의 변형)

| 원칙 | 설명 |
|------|------|
| 멈추지 않기 | 완벽주의 OFF, 일단 끝까지 쓰기 |
| 순서대로 | 1화부터 순서대로 (뒤로 돌아가지 않기) |
| 메모 활용 | 퇴고할 부분은 [TODO] 표시하고 넘어가기 |
| 일정한 분량 | 매일 최소 분량 설정 (ex: 3,000자/일) |

### 5.2 막힐 때의 대처법

| 상황 | 대처 |
|------|------|
| 전개를 모르겠다 | 캐릭터에게 "너라면 어떻게 하겠어?" 질문 |
| 쓰기 싫다 | 좋아하는 장면 먼저 쓰기, 후에 연결 |
| 이전 회차가 마음에 안 듦 | 퇴고에서 해결. 지금은 전진 |
| 소재가 바닥남 | 플롯 재검토, 새 서브플롯 도입 |

## 6. 5단계: 퇴고/자가 편집

### 6.1 퇴고 3회전 시스템

**1회전: 구조 퇴고 (큰 그림)**
- 플롯 구멍 없는가?
- 페이싱이 적절한가?
- 캐릭터 아크가 완성되는가?
- 복선이 모두 회수되는가?

**2회전: 장면 퇴고 (중간 그림)**
- 각 장면이 기능을 하는가? (기능 없으면 삭제)
- 장면 간 전환이 자연스러운가?
- 감정선이 연결되는가?
- 대화가 자연스러운가?

**3회전: 문장 퇴고 (세부)**
- 불필요한 단어 삭제
- 비유의 신선함 점검
- 종결어미/문장 시작 다양성
- 맞춤법/오탈자 점검

### 6.2 퇴고 체크리스트

**삭제 후보**:
- [ ] "매우", "정말", "너무" 등 강조 부사 → 행동으로 대체
- [ ] "~것 같다", "~인 듯 하다" → 단정으로 변경 가능?
- [ ] 같은 정보를 2번 이상 전달하는 문장 → 1개만 남기기
- [ ] 플롯에 기여하지 않는 장면 → 삭제 or 통합

**강화 후보**:
- [ ] Tell만 있는 감정 → Show로 전환
- [ ] 구체성 부족한 묘사 → 감각 디테일 추가
- [ ] 약한 오프닝/클리프 → 강화

## 7. 연재 특화: 버퍼 관리

### 7.1 이상적 버퍼

| 연재 상태 | 최소 버퍼 |
|----------|----------|
| 연재 시작 전 | 20~30화 |
| 연재 중 | 10화 이상 |
| 유료 전환 전 | 30화 이상 |

### 7.2 버퍼의 역할
- 퀄리티 관리 (퇴고 시간 확보)
- 독자 반응 대응 (수정/방향 전환 여유)
- 건강 관리 (체력/정신 부담 분산)
- 비상 대비 (병, 개인 사정)
