## 1. Overview

### 1.1 Purpose

크로마버스 웹사이트의 전체 UI를 대규모로 개편하여, 세계관의 시각적 정체성을 극대화하고 독자에게 몰입형 경험을 제공한다.

### 1.2 Design Decisions (확정)

| 항목 | 선택 | 설명 |
|------|------|------|
| **디자인 무드** | Cinematic Immersive | 영화 트레일러/게임 씬 느낌, 풀스크린 비주얼, 빛과 어둠의 연출 |
| **랜딩 페이지** | Scroll Cinema | 풀스크린 씬 스냅 스크롤, 각 섹션이 하나의 '씬'으로 전환 |
| **네비게이션** | Radial Menu + Scene Indicator | 고정 버튼 → 동심원 메뉴 + 우측 도트 인디케이터 |
| **테마 전략** | 빛(루미나스) vs 어둠(쿠로겐) | 세계관 연동 듀얼 테마, 각 모드 독립 색채 팔레트 |
| **세계관 허브** | Constellation Map | 12개 노드가 빛으로 연결된 인터랙티브 별자리 맵 |
| **세계관 상세** | Tab Chapters | 시각적 탭 네비 + 시네마틱 콘텐츠 전환 애니메이션 |
| **소설 리더** | Immersive Reader | 전체 화면, 장면 연동 배경 변화, glow 진행률 바 |
| **이미지 스타일** | Mixed Media | 세계관=디지털페인팅, 캐릭터=일러스트, 배경=환상적 풍경 |
| **푸터** | World Teaser | 매번 다른 세계관 조각/캐릭터 명언으로 재방문 유도 |
| **모바일** | 핵심 유지 + 절제된 인터랙션 | 복잡한 인터랙션 간소화, 핵심 비주얼/콘텐츠는 유지 |

### 1.3 Design Principles

1. **시각적 가시성 최우선** — 모든 텍스트/요소가 WCAG AA 이상 대비 보장
2. **인터랙티브 우선** — 정적 요소보다 마우스/스크롤 반응 요소 우선
3. **이미지 중심** — 아이콘/텍스트 장식 대신 생성 이미지 에셋 활용
4. **세계관 일체화** — UI 자체가 크로마버스 세계의 일부처럼 느껴져야 함
5. **듀얼 모드 철저 검증** — 빛/어둠 모드 각각 독립적으로 완성도 보장

### 3.2 네비게이션 구조

```
Radial Menu + Scene Indicator:

[고정 버튼] — 우측 하단 (모바일: 하단 중앙)
  └── 클릭 시 동심원 메뉴 펼침
      ├── 세계관 (노드 1, 색상: cognis/suppress)
      ├── 소설   (노드 2, 색상: ador/erode)
      ├── 소개   (노드 3, 색상: vitalis/distort)
      └── 빛↔어둠 토글 (중앙 또는 별도 노드)

[Scene Indicator] — 우측 세로 도트 (랜딩 페이지 전용)
  ├── ● Scene 1 (Hero)     ← active glow
  ├── ○ Scene 2 (세계관)
  ├── ○ Scene 3 (소설)
  ├── ○ Scene 4 (색채)
  └── ○ Scene 5 (CTA)

상단 바: 로고 + 테마 토글만 (미니멀)
  ├── 스크롤 시 투명화 (콘텐츠 위에 떠있는 느낌)
  └── 마우스 상단 호버 시 페이드인
```

```
├── 배경: 어둠 속 4인의 실루엣 이미지
├── 인용문: 카이의 기록 (타이핑 효과)
├── 캐릭터 힌트: 호버 시 실루엣 빛남
└── CTA: "이야기 시작하기" 버튼

Scene 4: 색채 인터랙티브 (100vh)
├── 배경: 7채널 컬러 그라데이션
├── 인터랙티브: 마우스 따라 색이 섞이는 효과
├── 설명: RGB/CMYK 체계 간단 소개
└── "더 알아보기" → 세계관 허브

Scene 5: CTA (100vh)
├── 배경: 프리즈마폴 풍경 이미지
├── 메인 CTA: "세계관 탐험하기" / "소설 읽기"
├── 통계: "700화의 대서사 / 9개 지역 / 4개 종족"
└── World Teaser: 랜덤 세계관 명언
```

### 3.4 세계관 허브 — Constellation Map

```
Canvas/SVG 기반 인터랙티브 맵:

노드 배치 (원형 + 계층):
├── 상단: 창세 (가장 위, 근원)
├── 2단: 종족, 힘의 체계
├── 3단: 사회, 종교
├── 4단: 지리, 역사
├── 5단: 경제, 성장
├── 하단: 용어, 크로마스톰, 특수존재

연결선:
├── 관련 노드 간 빛나는 선 (bezier curve)
├── 맥동 애니메이션 (pulse glow)
└── 호버 시 연결된 노드도 하이라이트

인터랙션:
├── hover: 노드 glow + 프리뷰 이미지 팝업
├── click: 해당 섹션의 Tab Chapters로 이동
├── drag: 맵 이동 (데스크톱)
└── pinch: 줌 인/아웃

사용자 연동 (로그인 시):
├── 잠금 노드: 아직 읽지 않은 스포일러 콘텐츠는 잠금 표시
│   ├── 잠금 상태: 노드 어둡게 + 자물쇠 아이콘 + "N화까지 읽으면 해제"
│   ├── 해제 상태: 노드 밝게 glow + 정상 접근 가능
│   └── 해제 애니메이션: 자물쇠 깨지며 빛 파티클 효과
├── 진행률 표시: 전체 맵에서 해제된 노드 비율 (X/12)
└── 비로그인 시: 모든 노드 접근 가능 (스포일러 경고만 표시)

모바일 적응:
├── 세로 리스트 형태로 변환
├── 각 항목에 대표 이미지 썸네일
└── 탭하면 바로 Tab Chapters로
```

### 3.5 세계관 상세 — Tab Chapters

```
레이아웃:
┌─────────────────────────────────────────┐
│ [◉창세][○종족][○힘][○사회][○종교]...     │ ← 스크롤 가능 탭 바
├─────────────────────────────────────────┤
│ ████ HERO IMAGE (섹션 테마) ████         │ ← 탭별 다른 이미지
├─────────────────────────────────────────┤
│                                         │
│  콘텐츠 영역                             │
│  ├── 이미지+텍스트 교차 배치              │
│  ├── 인터랙티브 요소 (차트, 맵, 시뮬레이터) │
│  └── 관련 섹션 링크                       │
│                                         │
└─────────────────────────────────────────┘

탭 전환 애니메이션:
├── 콘텐츠: 페이드아웃 → 페이드인
├── 히어로: 크로스페이드
├── 탭 바: 선택 탭 glow 이동
└── 배경 색조: 섹션 테마색으로 미세 전환
```

```
레이아웃:

기능:

├── 읽기 설정: 폰트 크기, 줄간격 조절 (우하단 톱니)
├── 화 이동: 좌우 스와이프 (모바일) / 키보드 화살표

├── 읽기 위치 자동 저장: 마지막 읽은 위치(화+스크롤) Supabase에 저장
├── 이어 읽기: 다음 접속 시 마지막 위치로 자동 이동
├── 읽은 화수 추적: 완독 시 자동 기록, 프로필에 통계 표시
├── 세계관 해제 알림: 새 화 완독 시 관련 세계관 정보 해제 토스트

└── 비로그인 시: localStorage로 읽기 위치만 임시 저장 (동기화 없음)
```

### 3.7 모드 전환 애니메이션

```

├── 어둠이 밀려나며 중앙에서 빛이 퍼짐
├── 배경이 깊은 남색 → 밝은 크림으로 0.6s transition
├── 텍스트 색상 동시 전환
└── accent 색 전환: RGB glow → CMYK ink
```

## 5. Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| 이미지 에셋 생성 병목 | 높음 | 각 Phase와 병렬로 이미지 생성. placeholder 이미지로 우선 구현 |
| 듀얼 모드 가시성 불일치 | 높음 | Phase 1에서 WCAG 대비 수치 확정, Phase 6에서 전수 스크린샷 검증 |
| 스냅 스크롤 모바일 호환 | 중간 | CSS scroll-snap + JS fallback, 모바일에서 일반 스크롤 대안 |
| Canvas 별자리 맵 성능 | 중간 | SVG 우선 시도, 복잡도 높으면 Canvas로 전환. 모바일은 리스트 |
| Static Export 제약 | 낮음 | 모든 인터랙션은 클라이언트 사이드. 서버 의존 없이 구현 |
| 잠금/해제 매핑 복잡도 | 중간 | 초기에는 단순 화수 기반 해제. 향후 세부 조건(특정 장면 등) 추가 가능 |
| 비로그인 사용자 경험 저하 | 중간 | 비로그인도 모든 콘텐츠 접근 가능 (스포일러 경고만). 로그인은 추가 혜택 |
