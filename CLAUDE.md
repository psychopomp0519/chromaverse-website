# 크로마버스 프로젝트 — Claude 작업 지침

## 프로젝트 개요
크로마버스(Chromaverse) 통합 프로젝트: 세계관 기반 웹소설 + 게임 설계 + Next.js 웹 플랫폼.
- 소설: 본편 700화/35권/7대막, 현재 1~11화 집필 완료 (archive-v1)
- 게임: 크로마주 — 소설 시점(프리즈마력 780년)으로부터 약 700년 후(1500년)
- 웹사이트: Next.js 15 + Supabase + Cloudflare Pages

| 파일 | 역할 | 우선순위 |
|------|------|---------|

| `project-assets/settings/01_크로마주_세계관설정집_v2_3_4.md` | 세계관 백과 — 창세/종족/힘/사회/종교/지리/역사/경제 + **부록A 확정 용어** + **부록B 금지 용어** | 세계관 정본 |

### 문서 간 관계
```
세계관설정집 (용어/설정 정본)
  ├── 마스터 문서 (집필 원칙 + 서사 설계)
  │     └── 레퍼런스 JSON (범용 패턴 보조)
  └── 게임개발설계서 (게임 시스템/연표)
```

- 집필 규칙 충돌 시: **마스터 문서 > 기타**

## 작업 모드

### 모드 1: 세계관/설정 점검

**진입**: 세계관 설정, 용어 체계, 시간선, 캐릭터 설정 검토 요청 시

**필수 참조**: 세계관설정집 + 마스터 문서 Part B (서사 설계)

**점검 4원칙**:
1. **일관성** — 설정집 ↔ 마스터 문서 ↔ 실제 원고 간 불일치 탐지
2. **완결성** — 누락된 설정, 미정의 용어, 빈 캐릭터 항목 식별
3. **논리성** — 세계관 내 모순, 시간선 오류, 인과관계 결함 검출
4. **추적성** — 떡밥/복선 현황, 레이 총합 변화, 위기 에스컬레이션 추적

**교차 검증 체크리스트**:
- [ ] 세계관설정집의 용어가 마스터 문서의 용어 도입 스케줄과 일치하는가
- [ ] 마스터 문서의 7대막 구조가 세계관설정집의 역사 연표와 정합하는가
- [ ] 게임설계서의 프리즈마력 1500년 설정이 소설 시점(780년)과 모순 없는가
- [ ] 27개 비밀의 공개 시점이 대막별 서사 구조와 일치하는가
- [ ] 캐릭터의 크로마 값/능력이 힘의 체계 규칙을 위반하지 않는가

**출력 형식**:
```
## 점검 결과
### 발견된 문제
1. [심각도: 높음/중간/낮음] 문제 설명
   - 위치: 파일명, 섹션
   - 상세: 구체적 불일치/누락 내용
   - 제안: 수정 방안

### 확인 완료 (이상 없음)
- 항목 목록
```

### 모드 2: 소설 집필

**진입**: 소설 집필/수정/검토 작업 시

**필수 참조**: 마스터 문서 전체 + 세계관설정집 (부록A/B)

**effort**: max

**워크플로우** (6단계, 순서 엄수):
1. 사전 자료 확인 (설정집/마스터문서/플롯/이전 회차)
2. 플롯 보충 (부족 시 사용자 승인 후 진행)
3. 떡밥/복선 사전 확인 (누적 목록 검증)
4. 소설 작성 (md 형식, `project-assets/novel/크로마버스_{N}화_{제목}.md`)
5. 자동 검증 3단계 (떡밥 검증 / 독자 관점 분석 / 자체 수정)
6. 최종 제출 → `node scripts/convert-chapters.mjs` 실행

**핵심 규칙**:
- 시점: 1인칭 관찰자(카이), 회고 기록
- 분량: 7,000~9,000자 (공백 포함), 미달 불가
- 문체: 중문 20~40자, 종결어미 3회 연속 금지, 복문 50%+
- 용어: 세계관설정집 부록A만 사용, 부록B 절대 불가
- 상세 규칙은 마스터 문서 Part A 참조

### 모드 2.5: 플롯 작성

**진입**: 화별 플롯 신규 작성/수정/정교화 시

**필수 참조**: 세계관설정집 + 마스터 문서 Part B (서사 설계) + 기존 플롯 파일 + 떡밥복선 현황표

**effort**: max

**핵심 원칙**: 플롯은 소설을 바로 집필할 수 있을 정도로 최대한 자세하게 묘사한다. 플롯을 읽는 것만으로 해당 화의 장면이 머릿속에 그려져야 한다.

**화별 플롯 필수 구성요소**:

1. **헤더**
   - `### {N}화: {제목}`
   - `**시간/장소**`: 시간대 흐름 + 구체적 장소 이동 경로
   - `**날씨/분위기**`: 날씨, 빛의 상태, 공기의 질감, 공간의 감각적 특성

2. **장면 단위 서술** (장면 1, 장면 2, ... / 핵심 장면 / 장면 전환)
   - **행동**: 캐릭터가 무엇을 하는지 구체적으로 (동작, 시선, 손짓까지)
   - **대사**: 핵심 대사는 따옴표로 직접 기술 — 캐릭터 말투/어감 반영
   - **내면 독백**: 카이의 사고 과정, 판단 근거, 감정 반응을 구체적으로
   - **관계 역학**: 캐릭터 간 태도 변화, 시선, 거리감, 호칭
   - **세계관 디테일**: 해당 장면에서 드러나는 설정 (도구, 건축, 에너지, 풍습 등)을 장면 속에 녹여서 서술
   - **분위기 전환**: 장면 내 긴장도/감정 변화의 곡선

3. **오감 힌트** (5감 중 해당 화에 적합한 것 3~5개)
   - 시각, 청각, 촉각, 후각, 미각 — 구체적 묘사 문장으로 제시
   - 소설 작성 시 직접 활용할 수 있는 수준의 표현

4. **태그 블록** (화 끝에 배치, 순서 고정)
   - `【용어】`: 이 화에서 처음 도입되는 용어 + 도입 방식 (일상 노출/사건/대사 등)
   - `【비밀】`: 복선/비밀 배치 — `[코드]` 포함 (M/H/C/S 번호)
   - `【사이다/고구마】`: 감정 유형 + 강도(소/중/대) + 구체적 장면/대사
   - `【서브】`: 서브플롯 요소, 캐릭터 관계 변화, 세계관 심화 요소
   - `【회고】`: 카이의 회고적 장치 사용 시 (미래 시점 서술)
   - `【훅】`: 화 끝 갈고리 — 다음 화로 이어지는 절단 지점. 구체적 장면/대사로 기술

**상세도 기준**:
- 핵심 장면은 소설 본문으로 옮길 때 뼈대가 그대로 쓰일 수 있는 수준
- 대사는 실제 대사 수준으로 작성 (요약이 아닌 직접 인용)
- 캐릭터의 미세한 행동 변화(시선, 손의 움직임, 호흡, 멈춤)까지 포함
- 장면 전환 시 연결 고리(시간 경과, 공간 이동, 감정 연결)를 명시
- 크로마 값, 에너지 변화 등 수치적 요소는 정확한 숫자로 기술

**금지 사항**:
- "~하는 장면이 있다" 식의 요약형 서술 금지 → 직접 묘사
- 캐릭터 감정을 "슬프다", "화난다" 등 추상적 단어로 요약 금지 → 행동/대사로 표현
- 오감 힌트 생략 금지
- 훅 없는 화 금지

**분량 규칙**:
- 1화당 플롯 최소 분량 제한 없음, 최대 제한 없음
- 축소 불가 — 정보가 부족한 플롯보다 과도하게 상세한 플롯이 낫다

**파일 형식**:
- `project-assets/plots/크로마버스_대막{N}_화별플롯_{M}권_{시작}-{끝}화.md`
- 파일 상단에 범례 + 카이 서술 태도 명시
- 권 구분선 후 중막 정보 (감정 곡선 포함)
- 파일 하단에 캐릭터 성장 지표 표

### 모드 3: 게임 설계

**진입**: 게임 시스템, 밸런스, 콘텐츠 설계 작업 시

**필수 참조**: 게임개발설계서 + 세계관설정집

**핵심 원칙**:
- 게임 시점(프리즈마력 1500년)은 소설(780년)보다 약 700년 후
- 소설에서 만들어진 세계 질서(코드 네트워크, RGB-CMYK 공존)가 게임의 기본 전제
- 소설 캐릭터(카이, 코다 등)는 게임에서 전설/역사 속 인물로 등장
- 용어는 세계관설정집 부록A 통일

### 모드 4: 웹사이트 개발

**진입**: 웹사이트 기능/UI 개발 시

**기술 스택**: Next.js 15 (App Router, static export) + Tailwind CSS v4 + Framer Motion + Zustand + next-themes + Supabase (Auth + DB) + Cloudflare Pages

**규칙**:
- UI 작업 전 이미지 예시를 보여주고 사용자 선택 받기
- 스크린샷 검증 시 Puppeteer 사용 (Playwright X)
- 부록A 확정 용어만 사용, RGB/CMYK 색채 체계 반영
- 배포: Cloudflare Pages (wrangler CLI)

## 공통 규칙

### effort 설정
- 소설 집필/수정, 세계관 점검: **max**
- 평소 작업: auto

### 용어 규칙 (모든 모드 공통)
- **확정 용어** (세계관설정집 부록A): 루미나스, 쿠로겐, 프리즈마폴, 이로몬, 갸쿠몬, 이로쥬, 사이도, 스플릿, 탈색, 프라이머리, 블렌드, 레스, 하쿠텐, 보더, 더딥, 코드, 컨덕터, 디소넌스, 광명회, 심묵교, 균형파, 반전, 엔지, 세이지, 메이치, 코지, 시지, 아오치
- **금지 용어** (세계관설정집 부록B): 라디언스, 잉크, 스캐터링, 크로마라인 등 — 절대 사용 불가

### 설정 변경 시 규칙
1. 세계관 설정 변경 → 세계관설정집 수정 + 마스터 문서/게임설계서 정합성 확인
2. 서사 구조 변경 → 마스터 문서 Part B 수정 + 35권 구성표/비밀 검증표 업데이트
3. 용어 변경 → 부록A/B 수정 + 전 문서 일괄 반영

### 스크린샷 검증 (Puppeteer)

웹사이트에 반영된 변경사항을 시각적으로 검증할 때 사용.

**스크립트**: `node scripts/verify-settings.mjs [mode]`
- `all` — 전체 페이지 (기본)
- `world` — 세계관 페이지만
- `novel` — 소설 페이지만
- `quick` — 랜딩 + 허브 3장만

**검증 절차**:
1. 기존 `screenshots/` 폴더 내 png 자동 삭제
2. Puppeteer로 대상 페이지 캡처 (1280x900, fullPage)
3. HTTP 상태 + 시각 결과 확인
4. 캡처된 이미지를 Read 도구로 확인하여 시각 검증

**사용 시점**:
- 소설 원고 반영 후 (`convert-chapters.mjs` 실행 후)
- 세계관 콘텐츠 변경 후 웹 반영 확인
- UI/디자인 변경 후

**주의**:
- 반드시 `npm run dev` (포트 3099) 실행 상태에서 사용
- 스크린샷 전 기존 파일 삭제 후 새로 캡처 (스크립트 자동 처리)
- Playwright 사용 금지 — Puppeteer + 시스템 Chromium(`/usr/bin/chromium`) 사용

### 커밋 규칙
- 커밋 전 변경사항 확인 필수
- 설정 문서 변경 시 어떤 문서가 영향받는지 명시
